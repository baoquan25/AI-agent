Execute any bash command inside the sandbox.
This is the most versatile tool — use it for anything that requires a terminal command.

**When to use:**
* Git operations: git clone, commit, push, pull, diff, log, status
* Package management: pip install, npm install, yarn add, cargo build
* Build tools: make, cmake, gcc, javac
* System commands: ls, cp, mv, chmod, curl, wget, tar, zip
* Docker commands: docker build, docker run
* Debugging: env, which, whoami, cat /etc/os-release
* Running scripts: bash script.sh, python script.py, node app.js

**Parameters:**
* command (required): The bash command to run
* working_dir (optional): cd to this directory before running
* timeout (optional): Max seconds to wait (default: 60)

**Examples:**
```bash
command="git status", working_dir="/home/daytona/workspace"
command="pip install requests pandas numpy"
command="python --version && pip list"
command="curl -sSL https://example.com/data.json -o /tmp/data.json"
```

**Output:** Returns stdout + stderr combined. Truncated at 50k chars for very large outputs.
